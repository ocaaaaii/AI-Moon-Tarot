## 一、概念定位

### 三扇門 vs 第四扇門

| 門 | 名稱 | 核心體驗 | 頻率 |
|---|---|---|---|
| 第一扇 | 月之塔羅店鋪 | 問事占卜，深度對話 | 有問題時 |
| 第二扇 | 月神神社 | 抽籤求指引 | 定期儀式 |
| 第三扇 | 月神天啟 | 閱讀神明的日常故事 | 休閒、追劇 |
| **第四扇** | **眾神之庭** | **日常互動、祝福、冥想、陪伴** | **每天** |

「眾神之庭」的核心差異：它不是「問事」，也不是「儀式」，而是**日常性的神明陪伴空間**。使用者來這裡不一定有問題，他們只是想被祝福、被看見、或只是想和某個神明聊聊今天。

### Feature 1：週神諭（遷移 + 擴充）
**現況：** 已建於 `/tarot/oracle`，三位神（Persephone / Eos / Nyx）  
**遷移目標：** 搬到 `/garden/oracle`

週神諭包含兩個子功能，在同一頁面以 Tab 或卡片切換：

#### 子功能 A：Pick-a-Card 神諭牌組
- 目前三位神主持，之後擴充至全部七位
- 每週輪換主持神明 + 主題方向
- 例：Cynthia「月光下的人生選擇」、Athena「把難題拆開來看」、Poseidon「現在的能量流向」
- 技術：`weeklyDecks.ts` 加入更多 `DeckTheme`，`getCurrentWeekIndex()` 自動輪換

#### 子功能 B：每週星座運勢（新增）
**主持神明：** Cynthia（月之女神，掌管天象與命運流向）  
**路由：** `/garden/oracle/horoscope`

---

**核心架構：爬取原始運勢 → Cynthia 重新詮釋 → 全星座一覽展示**

不是純 AI 生成，而是「真實占星內容 × Cynthia 語氣轉譯」的組合。

**第二步：爬蟲架構**

沿用現有 Agent 1（Scraper）模式，加入星座爬蟲：

```
/agents/horoscopeScraper.py        ← 新增
/wiki-horoscope/                   ← 新增資料夾，存放爬下來的原始內容
  └── 2026-W26-aries.md            ← 命名格式：年份-週數-星座
  └── 2026-W26-taurus.md
  └── ...（共 12 個）
```

**執行時機：** 每週一凌晨自動執行（可用現有 scheduled tasks 系統排程）  
**爬取內容：** 12 星座 × 本週運勢原文（整體、愛情、事業、健康）  
**Rate limiting：** 沿用 Agent 1 的 `time.sleep` 隨機延遲，保護來源伺服器

**第三步：Cynthia 轉譯 API**

爬到原文後，由 Cynthia 重新用她的語氣詮釋：

```
GET /api/garden/horoscope?sign=aries&week=2026-W26
Response: SSE streaming
```

**Prompt 策略：**
```
你是 Cynthia，月之女神。
以下是這週【牡羊座】的占星運勢原文：
---
{scraped_raw_content}
---
請用你的語氣，重新詮釋這段運勢給牡羊座的人聽。
不是翻譯，是用月之女神的視角轉述：篤定、溫柔、帶著宿命感。
不說「可能」「也許」——你說的是月亮看見的事實。
分成四個面向：整體能量 / 愛情 / 事業財運 / 本週一件事
每個面向 2-3 句，不加標題，段落之間空一行。
```

**快取策略：** 同一星座同一週只生成一次，結果存入 `/wiki-horoscope/2026-W26-aries-cynthia.md`，之後直接讀取，不重複呼叫 AI。

### Feature 2：顯化願力
**概念：** 由不同神明主持不同領域的顯化冥想，帶領 User 做肯定語（affirmation）+ 引導想像

#### 神明 × 顯化領域對應

| 神明 | 顯化領域 | 關鍵詞 |
|---|---|---|
| Persephone / 花音 | 愛情 · 自我價值 | 被愛、吸引力、感情豐盛 |
| Eos / 曉 | 事業 · 人生方向 | 成功、決斷、前進 |
| Helios / 陽真 | 自信 · 清醒 | 真實、不妥協、看清楚 |
| Poseidon / 潮 | 療癒 · 釋放 | 放鬆、流動、放下執念 |
| Athena / 祈織 | 思路 · 突破瓶頸 | 解法、拆解、智慧 |
| Nyx / 萬夜 | 潛意識 · 深層恐懼 | 看見、接納、黑夜的禮物 |
| Cynthia / 月乃 | 月相能量 · 整體祝福 | 宇宙、直覺、安排 |

#### 體驗流程
```
1. User 選擇顯化主題（愛情 / 財富 / 自信 / 療癒 / 突破...）
2. 對應神明出現，給出一段引言（AI 生成 × 20-30 秒閱讀）
3. 【呼吸引導】3-5 個呼吸步驟（純前端動畫，不需 AI）
4. 【肯定語】神明念出 5-7 句肯定語（AI 生成，根據 User 選的主題）
5. 【顯化畫面】User 在輸入框寫下「我想顯化的是：＿＿＿」
6. 神明給出最後的祝福語（AI 生成，回應 User 的願望）
7. 可截圖保存（使用現有 modern-screenshot 套件）
```

#### API 規格
```
POST /api/manifest
Body: { personaId: string, domain: string, userWish: string }
Response: SSE streaming（祝福語）
```

#### 技術重點
- 呼吸動畫：純 CSS / motion.react，不需 Three.js
- 肯定語由 AI 生成（不寫死），但有結構性 prompt 確保品質
- 截圖功能複用現有 modern-screenshot

### Feature 3：純聊天 · 好感累計系統
**概念：** User 去各個神明的「個人領地」純聊天，累積與該角色的關係，解鎖獨特劇情

#### 設計哲學
這不是占卜，也不是問事。**只是陪伴**。
User 可以說「我今天很累」「我去看了一場電影」「我失戀了」，神明以他的角色個性回應，像一個真正認識你的人。

#### 角色領地對應

| 神明 | 領地名稱 | 氛圍 | 場景描述 |
|---|---|---|---|
| 天城月乃 | 月光茶室 | 靜謐、神秘 | 月光灑落的和室，兩人對坐喝茶 |
| 東雲曉 | 黎明演武場 | 直接、有力 | 清晨的竹林道場，老將坐在石台上 |
| 日向陽真 | 烈陽殿廊下 | 犀利、反差萌 | 神社的廊下，他靠著柱子曬太陽 |
| 神樂祈織 | 智慧花園小徑 | 溫暖、好奇 | 下午的花園小徑，她邊走邊問你問題 |
| 汐見潮 | 潮音池畔 | 沉靜、包容 | 傍晚的水池邊，靜靜坐在你旁邊 |
| 天城花音 | 春之花圃 | 溫柔、陽光 | 早晨的花圃，她正在澆花 |

> 注意：此功能對應神社線的六位角色（不含 Nyx / 萬夜——她只在夜裡出現，可做特殊時間限定）

#### 好感累計系統（Affection System）

**儲存方式：** localStorage（`gardenAffection_${personaId}`）  
**累計單位：** 對話輪次  
**等級設計：**

| 等級 | 稱呼 | 解鎖內容 |
|---|---|---|
| Lv.0 初次相遇 | 陌生人 | 基礎聊天 |
| Lv.1 熟悉 | 常客 | 角色會記得你上次說的一件事（軟記憶）|
| Lv.2 親近 | 老朋友 | 解鎖角色的私密話題（個人背景故事片段）|
| Lv.3 知己 | — | 解鎖專屬結局對話（每個角色一段） |

**軟記憶實作：**  
每次對話結束後，前端把本輪的「用戶說的關鍵事件」存入 localStorage（最多保留最近 3 筆），下次對話時作為 context 注入 system prompt。

```typescript
// 範例：存入軟記憶
interface SoftMemory {
  personaId: string;
  entries: { date: string; summary: string }[]; // 最多 3 筆
}
```

#### API 規格
```
POST /api/garden/chat
Body: { personaId: string, messages: Message[], softMemory: SoftMemory }
Response: SSE streaming
```

#### System Prompt 策略
- 每個角色有獨立的「庭院聊天 prompt」（有別於占卜 prompt，更日常、更輕鬆）
- Prompt 注入 softMemory 讓角色「記得」你上次說的事
- 等級越高，prompt 中的「親密度指令」越深（可以更直接表達情感、更多私密分享）
